        ⛏ How to build an archive

        An archive is a plain JSON file with a sessions array. Each session is one sanitized QA note, log excerpt, or transcript. The decoder reads each session's domain to muster the matching agent — everything stays in your browser.

        1. Create a file, e.g. my-archive.json.
        2. Add a sessions array; give each entry a domain and some text.
        3. Use only the recognized domains below (others are ignored).
        4. Save, then drop it on the intake panel or pick it with Choose file.
        FieldRequiredWhat it is
        idnoyour reference for the session
        toolnowhere it came from (claude, codex, opencode…)
        domainyesQA domain — drives extraction (see below)
        titlenoshort human label
        textrecommendedthe sanitized session content

        Recognized domain values:

        etl graphql api data_quality bugs prompts

        Minimal example — copy, edit, save as .json:

        {
  "archive": "my-team-archive",
  "version": 1,
  "sessions": [
    {
      "id": "s-001",
      "tool": "claude",
      "domain": "etl",
      "title": "Airflow reconciliation",
      "text": "airflow dag failure, spark row count vs hive, reconciliation on parquet"
    },
    {
      "id": "s-002",
      "domain": "api",
      "text": "rest endpoint wrong http status, added contract test for openapi payload"
    }
  ]
}
