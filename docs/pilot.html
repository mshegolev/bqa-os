◆ The problem

QA knowledge is real, but it is scattered. Bug reports live in the tracker, regression notes in a wiki, debugging context in chat threads, and the prompts that unblocked a tricky test only exist in someone's terminal history. When people switch teams or the sprint moves on, that context is lost and the next engineer re-discovers the same bug from scratch.

ScatteredBug reports, regression notes, prompts and session logs sit in five different tools.
Lost on handoffQA context walks out the door when people change teams or projects.
Re-done workThe same API contract bug or ETL reconciliation gap gets re-investigated each quarter.

▶ The local CLI flow

The pilot runs on your machine. Three commands, nothing uploaded:

bqa discover→ bqa ingest2→ bqa build
  • bqa discover — find QA-relevant sessions and artifacts locally
  • bqa ingest2 — read the sanitized artifacts you selected
  • bqa build — produce the QA memory specs and workflows above

Want to feel it first? The browser demo runs the decode step entirely client-side on synthetic data.

🛡 Privacy — local-first by design
  • The flow runs on your machine. No backend in this repo collects or stores your data.
  • You decide what to share. Bring only sanitized artifacts — no secrets, no real customer records.
  • Public examples on this site are synthetic and generic, never real session logs.
  • This page is static GitHub Pages content. Submitting interest is a plain email or template you control.

What data do I need to prepare?

10–30 sanitized QA artifacts: bug reports, regression checklists, test notes, sanitized QA/debugging sessions, and any prompts that worked well. Quality beats volume.

What should I NOT send?

No secrets, credentials, tokens, or real customer records. No raw production logs. If an artifact can't be sanitized in a couple of minutes, leave it out — we'd rather have less data than risky data.

How long does the pilot take?

About two weeks, time-boxed. A 20-minute fit check first, then the build runs over the following days with a working session or two.

What does it cost?

The pilot is paid and scoped to a fixed fee — not an open-ended free trial. We share the current pilot price on the fit-check call so it matches your scope. (Pricing is a working hypothesis while we run the first pilots.)

Will this replace my QA engineers?

No. BQA-OS captures and reuses QA knowledge so your team moves faster — it does not run autonomous QA or replace engineers, and the MVP does not ship enterprise integrations.
